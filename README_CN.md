# The Seed - 面向游戏开发者的 LLM Agent 框架

## 项目概述

**The Seed** 是一个面向游戏开发者的开源框架，目标是：

- 让游戏可以 **通过一套统一的 Agent 接入协议**，把自己的状态 / 操作暴露给 LLM-Agent，让LLM操控游戏单位获取信息，进行行动和玩家交互；

- 在可控算力消耗下，让 LLM 与传统游戏 AI 形成互补，协同处理决策与执行。


---
## 设计理念

1. **游戏开发者为 Agent 预留一个清晰的入口**

   - 默认前提是：游戏愿意为 Agent 暴露一层“观察 / 动作 / 事件”的接口。
   - 框架尽量减少侵入性，让接入工作更像“挂一个模块”，而不是重写一遍游戏逻辑。
   - 希望开发者能在保持自己架构风格的前提下，把 Agent 接入做得轻量、可维护。

2.	LLM 做上层决策，游戏做执行
	 - LLM 擅长：局势理解、计划生成、策略推理、行为解释。
	 - 游戏原生 AI (行为树 / 状态机 / 规则系统) 擅长：寻路、微操、条件判断、帧级逻辑。
	 - The Seed 支持这种分工：  
   **LLM 产出意图 → 游戏负责落地执行。**  
    用最少的模型调用，完成最稳的行为表现。

3. **不局限玩法**

   - 提供一套可扩展的 Action / Observation / Tool 协议。
   - 不预设“攻击 / 采集 / 建造”等固定语义，让每个游戏根据自身玩法定义动作与信息结构。
   - 框架负责把这些定义整理成对 LLM 友好的 Prompt 结构和调用约定，方便不同游戏在同一框架下演化自己的 Agent。

4. **从接入体验出发逐步迭代**

   - 项目目前还在开发建设中
   - 优先关注：一个新游戏项目从“没接入”到“有 Agent 跑起来”，需要多少步骤、多少代码。
   - 通过少量真实项目，逐步打磨脚手架、示例、调试工具和最佳实践。
   - 希望最终呈现的是一套 **工程上好用、文档清晰、容易推广给同事的方案**

## 项目规划（Roadmap）


### Phase 0 – OpenRA 验证期（⏳ 进行中 约 45~60 天）

- 定义：面向 RTS 的基础 Agent 协议（观察 / 动作 / Tick / 事件）
- 完成一个 **“开箱即用的 OpenRA + Agent Demo”**
- 整理开发者文档：
  - 如何在 OpenRA 内接入 The Seed
  - 如何写一个简单的 LLM-Agent 控制一方阵营

### Phase 1 – 框架固化 & 文档化

- 抽象出与具体游戏解耦的 **Core SDK**
- 完善：
  - Agent 生命周期管理
  - Tick / 计划 / 记忆 / 日志
  - 与多种 LLM（或本地模型）的适配层

### Phase 2 – 多游戏接入 & 社区建设

- 支持第二个游戏接入（优先：策略 / 模拟经营）
- 形成“**游戏接入样板集合**”（Sample Integrations）
- 组织面向开发者的：
  - Hackathon / Game Jam
  - 线上分享 / Workshop

### Phase 3 – Agent 生态

- 提出一套 **Agent 描述规范**，方便：
  - 不同游戏之间分享 / 迁移策略
  - 社区贡献自己的 Agent 角色
